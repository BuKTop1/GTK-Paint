## ✅ Проект успешно исправлен и собран!

### Что было исправлено:

1. **Проблема с повторным определением глобальных переменных** - переменные определялись в PFL.h без `extern`, что приводило к их копированию в каждый объектный файл.

2. **Решение:**
   - Все глобальные переменные перемещены в новый файл `src/globals.c`
   - В `include/PFL.h` использованы только `extern` объявления (декларации)
   - В `src/main.c` удалены дублирующиеся определения

3. **Новая структура:**
   - `src/globals.c` - определения всех глобальных переменных
   - `src/main.c` - главная функция и обработчики событий
   - `include/PFL.h` - только внешние (extern) объявления
   - Другие исходные файлы используют extern из PFL.h

### Как собрать проект:

```bash
cd /home/buktop/projects/PFL

# Вариант 1: С использованием Makefile (рекомендуется)
make clean    # Очистить старые сборки
make          # Собрать проект

# Запустить приложение:
./build/PFL

# Вариант 2: С использованием CMake
mkdir cmake_build
cd cmake_build
cmake ..
make
./PFL
```

### Команды Makefile:

- `make` или `make release` - сборка с оптимизацией
- `make debug` - сборка с отладочной информацией
- `make clean` - удалить скомпилированные файлы
- `make run` - собрать и запустить
- `make debug-run` - запустить в GDB отладчике
- `make help` - показать справку
